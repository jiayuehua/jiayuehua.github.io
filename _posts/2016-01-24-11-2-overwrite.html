在linux操作中，常常遇到这种情况，想要改写某文件，方法一般是将原来的操作标准输出重定向到一个文件，然后使用cp或mv覆盖原始文件。比如

$ cat –n source.cpp >source2.cpp; mv source2.cpp source.cpp

有没有更好的方法呢？

答案是overwrite， 是一个特别精巧的shell脚本。见经典名著<>第112页，这里给出源码，也可以从https://github.com/jiayuehua/hoc/ 中下载misc/overwrite3 强烈建议每个接触类unix的朋友都有一本该书。

使用方法 ：

$ overwrite dstfile command

如

$ overwrite source.cpp cat -n source.cpp

$ overwrite source.cpp cat -n source.cpp |head

源码：

# overwrite: copy standard input to output after EOF

# final version

 

opath=$PATH

PATH=/bin:/usr/bin

 

case $# in

0|1)    echo 'Usage: overwrite file cmd [args]' 1>&2; exit 2

esac

 

file=$1; shift

new=/tmp/overwr1.$$; old=/tmp/overwr2.$$

trap 'rm -f $new $old; exit 1' 1 2 15    # clean up files

 

if PATH=$opath "$@" >$new        # collect input

then

    cp $file $old    # save original file

    trap '' 1 2 15    # we are committed; ignore signals

    cp $new $file

else

    echo "overwrite: $1 failed, $file unchanged" 1>&2

    exit 1

fi

rm -f $new $old
